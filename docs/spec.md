## 📚 The "Blue Tech" Knowledge Stack (Free & Open)

For students with non-technical backgrounds, prioritize visual and conceptual resources that explain *why* AI is used before *how* to code it.

| Resource Type | Recommended Free Source | Why it fits your context |
| :--- | :--- | :--- |
| **Foundational Textbook** | **"The Blue Economy 3.0"** by Gunter Pauli | Explains the "Zero Waste" philosophy of maritime innovation. |
| **AI Theory** | **Elements of AI** (University of Helsinki) | A world-class, free course designed specifically for non-math backgrounds. |
| **Maritime AI Research** | **MIT Sea Grant / AUV Lab** | Provides open access to papers and **real datasets** (sonar, lidar, and AIS data). |
| **Physical AI Guide** | **Microsoft’s "AI for Beginners"** | A curriculum on GitHub that uses visual metaphors to explain "Physical" agentic behavior. |

## 🛠️ The "Vibe Coding" Technology Stack

Since you are under-resourced, avoid paid subscriptions. Use these browser-native technologies that run on almost any laptop:

1.  **Vessel.js:** A specialized JavaScript library for conceptual ship design and simulation. It’s the maritime-specific version of Three.js.
2.  **Pyodide / WebAssembly:** Runs Python logic (like a small AI routing model) directly in the browser, so students don't need to install Python or manage servers.
3.  **Hugging Face (In-Browser):** Use `transformers.js` to run small object detection models (for identifying ships in video) entirely on the client side.
4.  **Wokwi:** A free, web-based simulator for Arduino and IoT hardware. Perfect for prototyping "Physical AI" sensors without buying physical parts.

## 🎓 How to Teach This (The "AI Bridge" Method)

For students at an under-resourced university, the goal is **AI Literacy** through creation, not syntax.

### 1. Start with the "Seattle Problem"
Ask students to visit the **Maritime Innovation Center** at Fisherman's Terminal (or the virtual 2026 tour). Have them identify one physical problem, like: *"How do we prevent boats from hitting whales without stopping commerce?"*

### 2. Research with "Search AI"
Have students use Gemini or Perplexity to find "Kaggle datasets for maritime ship detection" or "open-source sonar data." This teaches them that the *data* is the fuel, not the code.

### 3. Build via "Vibe Coding"
Instead of teaching them `const [state, setState] = useState()`, have them use Cursor to:
> "Make a 3D ocean in my project where a red dot appears whenever the sonar data detects an object."

### 4. Deploy and Demo
Students deploy to **GitHub Pages** (free). Their "final exam" is a public link they can show to the **Washington Maritime Blue** alliance members for internships.
